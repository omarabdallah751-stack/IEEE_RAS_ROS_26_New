Node:each node responsible for a specified modular purpose like a wheel motor and can send and receive data from another node

Topic:act like busses between nodes which take the data from the publisher node to the subscriber node

Service:the same as topics but, with a different method of communicating with(call-and-response model) but when only the service client call request "the data don't be given as continual updates"
